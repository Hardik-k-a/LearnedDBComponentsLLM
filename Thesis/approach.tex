This chapter presents the overall approach for creating and benchmarking learned database components using large language models and active learning. We introduce the system architecture at a conceptual level and describe the core ideas that motivate our pipeline design. The detailed algorithmic treatments of the query generation methodology and the training process with active learning are deferred to Chapters~\ref{chap:query_generation} and~\ref{chap:training_approach}, respectively. Experimental results including workload analysis, learning curves, and model evaluation are presented in Chapter~\ref{chap:experiments}.


\section{Benefits of the Proposed Framework}

The proposed framework offers several important benefits that distinguish it from prior work on learned cardinality estimation. First, the system is designed to be schema-agnostic: given a schema definition containing table names, column names, data types, and foreign key relationships, the system automatically generates appropriate queries, validates them against the schema, and produces correctly structured training data without any manual configuration. This means that the same pipeline can be applied to entirely different databases without modification, making it a general-purpose tool for training learned estimators.

Second, the framework is built around modular and extensible components. The query generation module can work with any LLM provider or fall back to a deterministic synthetic generator. The active learning module supports pluggable acquisition functions, and the model architecture can be swapped without affecting the rest of the pipeline. This modularity enables researchers to isolate and evaluate individual contributions independently.

Third, the system provides production-grade robustness through timeout handling for long-running database queries, automatic retry logic with exponential backoff for transient errors, and graceful degradation when queries fail after maximum retries. These mechanisms ensure that the pipeline can process thousands of queries without manual intervention, even when operating on complex schemas with expensive join queries.

Finally, the framework supports full reproducibility through configurable random seeds, complete experiment configuration logging, CSV-based results export, and timestamped output directories. These features are essential for scientific rigor and enable meaningful comparisons across different experimental configurations.

\section{Pipeline Overview}

The pipeline begins with query generation, where a large pool of unlabeled SQL queries is produced either by prompting a large language model with the database schema or by using a deterministic synthetic generator. Unlike traditional template-based generators that rely on predefined patterns, LLMs capture complex linguistic and structural patterns present in real-world workloads, producing queries with greater diversity in join topologies, predicate combinations, and selectivity profiles. The details of the prompt engineering strategy, diversity mechanisms, and the multi-layer validation pipeline that filters invalid LLM output are presented in Chapter~\ref{chap:query_generation}.

Once a pool of valid queries has been assembled, the pipeline proceeds to database labeling, where ground truth cardinalities are obtained by executing each query against the target database. This is the most computationally expensive stage of the pipeline, and it is precisely this cost that motivates the use of active learning. Rather than labeling the entire query pool, only a strategically chosen subset is executed, dramatically reducing the total computation required.

Before queries can be fed to the neural network, they must be transformed into fixed-size tensor representations. The feature extraction stage encodes each query as three sets of vectors representing tables, predicates, and joins. A key element of this encoding is the bitmap feature, which captures the selectivity of each query's filter predicates on a pre-sampled subset of the underlying data. These bitmaps provide the model with data-aware signals that go beyond purely structural features, enabling it to learn correlations between query predicates and actual data distributions.

The active learning loop forms the core optimization mechanism of the pipeline. It operates iteratively: an initial batch of queries is randomly selected and labeled to bootstrap the model, after which the model is trained and then used to score all remaining unlabeled queries according to an acquisition function. The highest-scoring queries, those where the model is most uncertain or where the greatest learning potential exists, are selected for labeling and added to the training set. This cycle repeats for a configurable number of rounds, progressively building a training set that is both compact and highly informative. The framework supports multiple acquisition strategies including random sampling, uncertainty sampling, and Monte Carlo dropout, each offering different trade-offs between computational cost and sample efficiency. The complete training methodology, MSCN architecture, and active learning strategies are detailed in Chapter~\ref{chap:training_approach}.

The final stage produces comprehensive evaluation metrics and visualizations including Q-error distributions, learning curves, predicted-versus-actual scatter plots, and workload analysis dashboards. These outputs enable detailed assessment of both the generated workload quality and the trained model's estimation accuracy.


\section{Summary}

This chapter introduced the overarching approach for creating and benchmarking learned database components using LLMs and active learning. We described the benefits of the proposed framework, presented the six-stage pipeline architecture, and outlined the flow of data from schema ingestion through query generation and validation to active learning and model evaluation.
